# D_GPUaaS-v2 프로젝트 가이드

## 푸시 요청 처리

사용자가 "v2.6으로 푸시해줘", "깃헙에 올려줘" 등 버전 푸시를 요청하면 아래 순서로 실행한다.

### 1. 버전 확인
메시지에서 버전명 추출 (예: v2.6, v2.7)

### 2. Changelog 작성
`git diff HEAD~1 HEAD` 분석 후 변경 내용을 실제로 작성해서 changelog 파일에 삽입한다.

**파일 경로**: `../docs/changelog_{버전}_{YYMMDD}.md`
- 파일이 있으면 파일명 날짜를 오늘로 갱신
- 파일이 없으면 템플릿으로 새로 생성

**테이블 형식**:
```
| No. | 버전 | 변경 유형 | 변경 항목 | AS-IS | TO-BE | 변경 일시 |
| --- | ---- | ---------- | --------- | ----- | ----- | -------------------- |
```

**작성 규칙**:
- No.: 현재 최댓값+N부터 내림차순으로 위에 삽입 (큰 번호가 위)
- AS-IS / TO-BE: 플레이스홀더 금지. diff에서 실제 내용 작성
- 신규 추가 항목의 AS-IS: `- -`
- AS-IS와 TO-BE의 불렛 수 대칭 유지
- 화면 위치는 AS-IS 불렛 앞에 `[화면명]` 으로 표기
- 변경 일시: `2026-07-24&nbsp;15:46` 형식

**변경 유형 판단 기준**:
| 유형 | 기준 |
|------|------|
| `UX&nbsp;정책` | 화면 레이아웃·구조 변경 |
| `추가` | 신규 기능·UI·콘텐츠 |
| `변경` | 기존 기능 동작 수정 |
| `제거` | 기능·콘텐츠 삭제 |
| `용어` | 텍스트·번역·i18n 변경 |
| `서비스&nbsp;정책` | 정책·규칙 적용 |

### 3. Git 태그 강제 푸시
```bash
git tag -f {버전}
git push origin {버전} -f
```

---

## Changelog 파일 규칙

- 버전별 파일 하나 유지: `changelog_v2.6_260724.md`
- 구분선 너비: 변경 유형 `----------`, 변경 일시 `--------------------`
- 다단어 유형은 &nbsp; 사용: `서비스&nbsp;정책`, `UX&nbsp;정책`
